Installing GitHub Copilot CLI (Laptop Setup Only)

‚ö†Ô∏è Note: These instructions are for laptop setup only. Instructions for Biowulf (HPC) will follow separately.

GitHub Copilot CLI brings AI assistance directly to your terminal ‚Äî helping you explain, generate, and optimize shell commands, scripts, or code snippets. It acts like an AI command-line companion, understanding natural language queries and producing executable shell syntax. This guide describes how to install both the GitHub CLI extension (gh copilot) and the standalone npm Copilot CLI (with ??-style aliases) within a single, unified Mamba environment.

üß© Prerequisites

Before proceeding, ensure you have the following:

  • ‚úÖ A GitHub account with Copilot CLI access (available via your GitHub Enterprise account)
  • ‚úÖ Mamba or Conda installed locally
  • ‚úÖ A terminal configured for bash or zsh
  • ‚úÖ Internet connectivity (Copilot CLI requires an active connection to GitHub servers)

üß± Step 1. Create and Activate a Unified Environment

We‚Äôll create a single environment to house both the GitHub CLI (gh) and the Node.js-based Copilot CLI. Using one environment avoids path conflicts and keeps both tools version-synced. You may also use an existing mamba environment on your laptop ‚Ä¶ remember to activate it first.

mamba create -n ghcopilot -c conda-forge gh nodejs -y
mamba activate ghcopilot

Verify Installation

gh --version
node --version
npm --version

Expected output (approximate):

gh version 2.63.0 (2025-03-15)
node v20.11.0
npm 10.2.5

üß† Tip: Using Node.js ‚â• 18 ensures ES2022 compatibility and smoother npm package builds.

üîë Step 2. Authenticate GitHub CLI

Authenticate GitHub CLI to your personal or enterprise account:

gh auth login

When prompted:

  • Choose GitHub.com
  • Select HTTPS as the protocol
  • Opt to Authenticate Git with your GitHub credentials
  • Choose Login with a web browser and follow the browser flow ‚Ä¶ you may be given a code in the terminal which you should be copy-pasting in the browser

Confirm authentication:

gh auth status

Expected output example:

‚úì Logged in to github.com as <your_GH_handle> (GitHub Enterprise)
‚úì Git operations authenticated for github.com
...

‚öôÔ∏è Step 3. Install Copilot CLI as a GitHub Extension

The GitHub CLI supports extensions that integrate seamlessly. Copilot CLI is one such extension.

gh extension install github/gh-copilot

Check the installation:

gh extension list

You should see:

github/gh-copilot  latest  Copilot for gh CLI

Now, test the basic commands:

gh copilot --help

  copilot [command]

Examples:

$ gh copilot suggest "Install git"

Use "copilot [command] --help" for more information about a command.

üß† Step 4. Install Standalone Copilot CLI (via npm)

While the gh extension is GitHub-integrated, the standalone Copilot CLI offers a lightweight, shell-native interface. It provides convenient commands such as ??, gh?, and git?, which allow you to quickly generate shell commands, GitHub CLI commands, or git commands based on natural language prompts. This makes it easier to automate tasks, discover command-line workflows, and boost productivity directly from your terminal without leaving your development environment.

Install it inside the same environment:

npm install -g @githubnext/github-copilot-cli

Check installation path and version:

which github-copilot-cli
github-copilot-cli --version

Expected output should be something like this:

~/mambaforge/envs/ghcopilot/bin/github-copilot-cli
GitHub Copilot CLI version 1.0.8

üîë Step 5. Authenticate Standalone Copilot CLI

Login to associate your CLI instance with your GitHub account:

github-copilot-cli auth login

This launches a browser window to complete authentication. Again, copy-paste the code in browser.

Confirm login status:

github-copilot-cli auth status

Expected output:

‚úì Authenticated as <your_github_handle>
...

üß∞ Step 6. Enable Shell Aliases (??, gh?, git?)

To enable shortcut-style commands, append the following to your shell configuration file (~/.bashrc or ~/.zshrc):

eval "$(github-copilot-cli alias -- "$0")"

Reload your shell:

source ~/.zshrc

Verify alias activation:

alias | grep copilot

You should see entries such as:

alias ??='github-copilot-cli what-the-shell'
alias gh?='github-copilot-cli git-assist'
alias git?='github-copilot-cli explain'
